# Git 停止跟踪 build/ 目录操作指南

## 场景说明

项目在提交时已包含 `build/` 目录及其文件，现需让 Git 停止跟踪该目录，并在后续提交中将其从远程仓库删除，同时保留本地文件。

---

## 操作步骤

### 1. 从 Git 索引中移除（保留本地文件）

```bash
git rm -r --cached build/
```

| 参数 | 说明 |
|------|------|
| `rm` | 移除文件 |
| `-r` | 递归删除（针对文件夹） |
| `--cached` | **关键参数**，仅从 Git 暂存区移除，保留本地物理文件 |

---

### 2. 配置 .gitignore

在项目根目录的 `.gitignore` 文件中添加：

```
/build/
```

> 使用 `/build/` 而非 `build/`，确保仅匹配根目录下的 build 文件夹，避免误匹配其他路径下的同名目录。

---

### 3. 提交并推送

```bash
git add .gitignore
git commit -m "chore: 停止跟踪 build 目录，并更新 .gitignore"
git push
```

执行后，远程仓库中的 `build/` 目录将被删除，后续提交不再包含该目录。

---

## 协作者注意事项

当其他协作者执行 `git pull` 后，其本地 `build/` 文件夹**不会被自动物理删除**，仅不再被 Git 跟踪。如需清理本地未跟踪的 `build/` 目录，可手动删除或执行：

```bash
git clean -fd
```

> ⚠️ `git clean -fd` 会删除所有未被 Git 跟踪的文件和目录，请确认无误后使用。

---

## 特殊情况提醒

如果 `build/` 目录中包含**密码、密钥、凭证**等敏感信息，上述操作**不够安全**，因为敏感数据仍存留于 Git 历史记录中。此时需使用 `git filter-repo` 或 `BFG Repo-Cleaner` 重写历史以彻底清除。
